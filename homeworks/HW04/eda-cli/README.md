# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Просмотр первых/последних n-строк датасета:

Просмотр первых 10 строк датасета:

```bash
uv run eda-cli head data/example.csv --n 10
```

Просмотр последних 10 строк датасета:

```bash
uv run eda-cli tail data/example.csv --n 10
```

Параметры:
- `--n` – количество строк (по умолчанию `5`) 
- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

Параметры:

- `--title` – заголовок отчёта (по умолчанию `EDA-отчёт`);
- `--out-dir` – каталог для отчёта (по умолчанию `reports`);
- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).
- `--max-hist-columns` – максимум числовых колонок для гистограмм (по умолчанию `5`);
- `--min-missing-share` – минимальная доля пропущенных значений (по умолчанию `0.5`);
- `--min-duplicates-share` – минимальная доля дубликатов (по умолчанию `0.2`);
- `--min-zeros-share` – минимальная доля нулевых значений (по умолчанию `0.9`);
- `--max-columns` – максимум категориальных колонок для top-категорий (по умолчанию `5`);
- `--top-k-categories` – количество top-значений для категориальных признаков (по умолчанию `5`);

Пример использования дополнительных параметров:

```bash
uv run eda-cli report data/example.csv --title "REPORT" --out-dir reports_example --min-missing-share 0.8 --min-duplicates-share 0.3 --min-zeros-share 0.9 --max-hist-columns 5
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `zeros.csv` – нулевые значения по колонкам;
- `duplicates.csv` – дубликаты по строкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `zeros_matrix.png` – визуализация нулевых значений;
- `correlation_heatmap.png` – тепловая карта корреляций.
